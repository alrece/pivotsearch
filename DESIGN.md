## 设计哲学

本地搜索工具的 UI 应该**快、安静、专注**：

- **快**：输入即响应（<100ms 出结果），无加载态闪烁
- **安静**：不抢主角，搜索结果才是主角；中性色为主，强调色克制
- **专注**：一个搜索框统领全局，结果清晰可扫，预览不打断阅读

## 色彩系统（初稿，Phase 4 细化）

```css
:root {
  /* 主色：沉稳的靛蓝，传递"可靠、精确检索" */
  --ps-primary: #3b5bdb;
  --ps-primary-hover: #4263eb;

  /* 中性色：暖灰，长时间阅读不刺眼 */
  --ps-bg: #fafafa;
  --ps-surface: #ffffff;
  --ps-text: #1a1a1a;
  --ps-text-secondary: #6b6b6b;
  --ps-border: #e5e5e5;

  /* 功能色 */
  --ps-highlight: #fff3bf;    /* 命中高亮背景 */
  --ps-highlight-text: #1a1a1a;

  /* 深色模式（预留） */
  --ps-bg-dark: #1a1a1a;
  --ps-surface-dark: #252525;
}
```

## 字体

- 界面：系统默认（-apple-system / Segoe UI / sans-serif）
- 搜索结果正文：等宽友好的比例，确保代码片段可读
- 高亮：加粗 + 背景色，不改变文字颜色

## 组件约定

- 基于 Element Plus，只覆盖必要的主题变量
- 搜索框：大号、居中或顶部，Enter 触发精确搜索，输入 debounce 200ms 触发即时搜索
- 结果列表：虚拟滚动（万级结果不卡），每项显示 文件名 + 命中片段（高亮）+ 路径 + 大小 + 修改时间
- 预览面板：右侧可折叠，点击结果项展开，重新解析原文件渲染（与索引内容解耦）
- 索引管理：设置页，列表式展示已添加的索引根 + 添加/删除/重建按钮

## 交互规范

- 即时搜索：输入 200ms 后自动搜，结果区显示加载骨架
- 高亮：命中关键词在结果列表和预览面板都用 `--ps-highlight` 背景标记
- 快捷键：`Ctrl/Cmd+K` 聚焦搜索框，`Esc` 清空，`Ctrl/Cmd+Enter` 精确搜索
- 空态：无结果时显示友好提示 + 搜索技巧
